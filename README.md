## 📌 About

Most competitive intelligence tools in pharma cost thousands per month and are locked behind enterprise paywalls (Cortellis, Kognitic, Citeline). This project replicates the core workflow using only public data, the same sources a biotech analyst pulls up on day one.

The dashboard answers three questions analysts ask every day:

> *Who is competing in this indication? Who has the strongest data? What happens next?*

## 🎯 Momentum Score — How It Works

Each drug is scored on a 0–100 composite metric replicating how analysts rank pipeline assets:

```
Momentum Score = Phase Score (0–40)
               + Designation Score (0–25)
               + Risk Score (0–30)
               + Readout Proximity Bonus (0–5)
```

| Component | Logic |
|---|---|
| **Phase Score** | Approved = 40 pts · Phase III = 28 · Phase II = 16 · Phase I = 8 |
| **Designation Score** | Breakthrough Therapy = 25 · Fast Track = 18 · Orphan Drug = 15 |
| **Risk Score** | Low risk = 30 pts · Medium = 18 · High = 8 |
| **Readout Bonus** | Readout within 12 months = +5 · within 24 months = +3 |

> Calculated in Python — not hardcoded. Recalculates every Monday from live API data.

## 🤖 ETL Pipeline — How The Data Flows

```
Every Monday 9am UTC
        ↓
GitHub Actions triggers fetch_pipeline_wars.py
        ↓
Python hits ClinicalTrials.gov API v2
for all 24 NCT IDs (requests library)
        ↓
Pandas cleans + structures raw JSON response
        ↓
Python calculates:
  - Final phase (API phase vs known approvals)
  - Risk score (phase + designation + program history)
  - Momentum score (weighted composite formula)
  - Indication-level analysis (avg momentum, phase counts)
  - Cross-indication analysis (company dominance, upcoming readouts)
        ↓
Outputs clean data.json
        ↓
GitHub Actions auto-commits data.json to repo
        ↓
Dashboard reads fresh data on next page load
```
